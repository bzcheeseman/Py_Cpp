PyObject* pycpp::make_dict ( int  num,

inline

Makes a PyDict.

Makes a python dictionary for the purpose of passing keyword arguments. Note that the order of arguments is as follows: make_dict(2*n, pycpp::to_python("key_n"), pycpp::to_python(<value_n>)) where n is the total number of key-value PAIRS.

Parameters
numNumber of arguments to the function
...Variable number of arguments of type PyObject *
Returns
A PyDict that can be passed directly to the py_module operator() call.

Makes a PyTuple.

Makes a python tuple for the purpose of passing arguments to a python function. All arguments to this function MUST be of type PyObject * as follows: make_tuple(2, pycpp::to_python(<value_1>), pycpp::to_python(<value_2>))

§ python_home

std::string pycpp::python_home = PYCPP_PY_HOME
static

Sets the python home. Generally scripts will be in the directory one up from build, so this is the default value. However this is easily changed by calling pycpp::python_home = "path/to/script".

§ which_python

std::string pycpp::which_python = PYCPP_WHICH_PYTHON
static

Selects the python executable - this is important to choose right because the wrong choice can have grave consequences - a.k.a. things won't work. If you're having trouble I recommend changing this first, especially if you used homebrew to get another python distribution alongside your system python.
